polyfill是什么

polyfill是一个js库，主要抚平不同浏览器之间对js实现的差异。比如，html5的storage(session,local), 不同浏览器，不同版本，有些支持，有些不支持。polyfill帮你把这些差异化抹平，不支持的变得支持了

数组的那些方法会改变原数组

  • 1、shift：将第一个元素删除并且返回删除元素，空即为undefined
  • 2、unshift：向数组开头添加元素，并返回新的长度
  • 3、pop：删除最后一个并返回删除的元素
  • 4、push：向数组末尾添加元素，并返回新的长度
  • 5、reverse：颠倒数组顺序
  • 6、sort：对数组排序
  • 7、splice:splice(start,length,item)删，增，替换数组元素，返回被删除数组，无删除则不返回
  • 8、fill:用一个元素填充原来的数组
  • 9、copyWithin:方法浅复制数组的一部分到同一数组中的另一个位置，并返回它，不会改变原数组的长度

JS什么情况下会出现内存泄漏

现代浏览器都使用了标记-清除垃圾回收算法。所有对JavaScript垃圾回收算法的改进都是基于标记-清除算法的改进。
此算法的大致步骤：

  • 1、设置一个叫做根（root）的对象（在Javascript里，根是全局对象）。垃圾回收器将定期从根开始，找所有从根开始引用的对象，然后找这些对象引用的对象
  • 2、垃圾回收器将所有可以获得的对象（活动对象）和所有不能获得的对象（垃圾对象）做出标记
  • 3、垃圾回收器释放垃圾对象所占用的内存并将其返回给操作系统

WebSocket SocketJS Stomp三者的关系

WebSocket是H5规范给出的网络层协议，旨在提供浏览器与服务端建立全双工的通信方式

SocketJS 由于WebSocket协议并不是所有浏览器都提供支持，所以又出现了一个SockJs。SockJS 是 WebSocket 技术的一种模拟，提供了’类Websocket’、一致性、跨平台的API，旨在浏览器和服务器之间创建一个低延迟、全双工、支持跨域的实时通信信道，主要特点就是仿生Websocket，它会优先使用Websocket作为传输层，在不支持WebSocket的环境回退使用其他解决方案，例如XHR-Stream、轮询。

Stomp：面向消息的简单文本协议，来为浏览器 和 server 间的通信增加适当的消息语义

DocType的作用，什么是混杂模式和严格模式

声明位于文档中的最前面，处于标签之前。告知浏览器的解析器，用什么文档类型 规范来解析这个文档。

严格模式：又称标准模式，是指浏览器按照 W3C 标准解析代码
混杂模式：又称怪异模式，是指浏览器用自己的方式解析代码。

如何区分：
1、 如果文档包含严格的 DOCTYPE ，那么它一般以严格模式呈现
2、包含过渡 DTD 和 URI 的 DOCTYPE ，也以严格模式呈现，但有过渡 DTD 而没有 URI，会导致页面以混杂模式呈现
3、DOCTYPE 不存在或形式不正确会导致文档以混杂模式呈现
4、HTML5 没有 DTD ，因此也就没有严格模式与混杂模式的区别，HTML5 有相对宽松的语法，实现时，已经尽可能大的实现了向后兼容

解构赋值的原理

解构赋值内部是将源数据使用函数ToObject将源数据转换成对象，然后引用对象中的属性，也就是说在解构的上下文中，原始值会被当成对象。这样意味着null和undefined不能被解构，否则会抛出异常

css中link和@import的区别

1、@import是css提供的语法规则，只能导入样式表；link是HTML提供的标签，不但可以加载css，还可以定义 RSS、rel 连接属性
2、页面加载时，link会同时被加载，而@import引入的css会等到页面全部下载完成再被加载
3、@import是css2.1定的语法，只在IE5以上才会被识别；link是html标签，不存在兼容问题
4、可以通过 JS 操作 DOM ，插入link标签来改变样式，@import无法使用此方式
5、link引入的样式权重大于@import引入的样式

position：absolute和float的区别

共同点：都可以使元素脱离文档流
不同点：float仍会占据位置，absolute会覆盖文档流中的其他元素

display: none 和 visibility: hidden区别

display 隐藏对应元素 元素所占据的空间依然存在
visibility 隐藏对应元素 元素不占空间

其他： 透明度为0 也可以隐藏元素

rgba 和 opacity 透明效果有什么不同

都能实现透明效果

opacity作用于元素，并且会影响元素内（子元素）的透明度，会被继承
rgba作用于元素，只影响元素的透明度，不会影响子元素，不会被继承

网页字体是使用奇数还是偶数呢

使用偶数

  • 设计师的原因 ps 提供的字号是偶数，自然到了 前端那边也是用的是偶数
  • 系统原因 Windows 自带的点阵宋体（中易宋体）从 Vista 开始只提供 12、14、16 px 这三个大小的点阵
  • 编程习惯 偶数更加容易计算 12 * 1.5
